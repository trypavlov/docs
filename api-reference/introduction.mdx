<Note>
  The Trig API allows you to programmatically interact with your customer data, campaigns, and lifecycle stages. Contact your Trig implementation team for API access.
</Note>

## Overview

The Trig API enables you to:

- **Read customer data** — Access organisation and people attributes
- **Trigger campaigns** — Programmatically add customers to campaigns
- **Update attributes** — Set attribute values from external systems
- **Receive webhooks** — Get notified when customers complete campaigns or behaviours

## Authentication

All API endpoints are authenticated using Bearer tokens.

```bash
curl -X GET https://api.trig.com/v1/organisations \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

## Getting Started

To get API access:

<Steps>
  <Step title="Contact your implementation team">
    Request API credentials from your Trig contact
  </Step>
  <Step title="Generate an API token">
    Create a token in Settings > API
  </Step>
  <Step title="Test your connection">
    Make a test request to verify authentication
  </Step>
</Steps>

## Webhooks

Trig can send webhooks when customers:
- Enter a campaign
- Complete a campaign
- Exit a campaign
- Complete a behaviour
- Enter or exit a lifecycle stage

Configure webhooks in **Settings > Integrations > Webhooks**.
